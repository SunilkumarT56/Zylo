# Security Policy

Security is taken seriously in this project.

## Reporting a Vulnerability

If you discover a security issue, please **do not open a public issue**.

Instead:

- Report it privately to the maintainers
- Provide a clear description of the vulnerability
- Include steps to reproduce if possible

## What to Include

Please include:

- Affected component or service
- Potential impact
- Suggested mitigation (if any)

## Response Process

- We will acknowledge the report
- Investigate the issue
- Release a fix as soon as possible
- Credit the reporter when appropriate

## Supported Versions

Only the latest version of the project is actively supported for
security updates.
